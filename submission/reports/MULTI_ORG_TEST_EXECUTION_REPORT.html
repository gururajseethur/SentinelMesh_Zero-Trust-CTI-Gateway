1. Test Objectives

  1. Validate that indicators from multiple organisations can be ingested and normalised into a common STIX 2.1 format.
  2. Verify cross-organisation correlation: the same threat IOC submitted by different partners is detected and linked.
  3. Confirm TLP policy enforcement: RED indicators are not visible to unauthorised sharing groups.
  4. Validate trust scoring: organisations with higher historical accuracy receive higher reliability scores.
  5. Confirm that YARA rules and network indicators can co-exist in the same bundle.

8. Recommended Actions (from TISP analysis engine)

  1. Promote multi-source high-confidence indicators to MISP sharing groups.
  2. Create TheHive cases for CRITICAL indicators tagged c2, ransomware, or credential-access.
  3. Push sanitized STIX bundles to TAXII collections for partner consumption.
  4. Review low-confidence single-source indicators before external sharing.

9. Lessons Learned

  • Subnet anonymization is effective: IPv4 addresses are grouped to /24 subnets in STIX output, preserving threat intelligence value while reducing PII exposure.
  • Multi-source confirmation is a strong signal: The 22 correlated indicators, especially those confirmed by all 3 orgs, represent the highest-priority actionable intelligence.
  • YARA rules integrate cleanly: All 5 YARA indicators normalised correctly to STIX with pattern_type: yara.
  • TLP:RED discipline: 34% of indicators are marked RED — reflects mature opsec culture where sensitive IOCs are protected.
